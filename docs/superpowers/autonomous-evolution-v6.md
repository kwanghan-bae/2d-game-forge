# 자율진화 게임 개발 프로토콜 v6

## v5 에서 배운 것

v5 는 7 에이전트 평가 + 통합 증명을 강제했으나, 실행 시 오케스트레이터가
**EncounterEngine.ts 단일 파일에 상수+곱셈기 추가** 패턴으로 고착되어
560+ 메카닉을 3500줄에 집어넣었다.

**결과**:
- ATK 곱셈기 70+개, EXP 72+개, Gold 67+개 (multiplier soup)
- 플레이어 의사결정 = 0 (모든 메카닉 자동 발동)
- UI/비주얼/사운드 변경 = 0 (엔진 내부만 변경)
- 게임비평가 재미 점수 3/10

**근본 원인**: "사이클 = 메카닉 1개 추가"로 정의한 것이 문제. 메카닉 추가는
가장 쉬운 작업이라 오케스트레이터가 항상 그것만 선택.

| 항목 | v5 | v6 |
|------|----|----|
| 사이클 정의 | 메카닉 1개 추가 | **레이어 순환 작업 1건** |
| 작업 범위 | EncounterEngine only | **5 레이어 강제 순환** |
| 곱셈기 추가 | 무제한 | **금지** (상한 도달) |
| 자동발동 메카닉 | 허용 | **금지** (choice 필수) |
| 파일 크기 | 무제한 | **800줄 상한** |
| 평가 주기 | 3사이클마다 | **10사이클마다** |

## 강제 순환 규칙

**연속 제한**: 동일 레이어 **2회 연속 금지**. 반드시 다른 레이어로 전환.

**10사이클 예산** (매 10사이클 체크):
- 구조: 최소 1회
- 시스템: 최대 3회
- UI/UX: 최소 2회
- 비주얼/사운드: 최소 2회
- 데이터/밸런스: 최소 1회

예산 미달 시 → 다음 10사이클에서 부족 레이어 우선 강제 배정.

---

## 사이클 구조

### 매 사이클 (소형 — 10-15분)

1. **레이어 선택**: 순환 규칙에 따라 이번 사이클 레이어 결정
2. **작업 결정**: 해당 레이어에서 가장 임팩트 있는 1건
3. **구현**: 코드 작성 + 테스트
4. **검증**: typecheck + test (변경 범위만)
5. **커밋**

### 매 10사이클: 에이전트 평가

7 에이전트 전원 dispatch:
- `game-critic` → 5축 점수
- `level-designer` → 밸런스 진단
- `game-planner` → 다음 10사이클 우선순위
- `ui-ux-designer` → UI 개선점
- `qa-engineer` → 회귀 위험
- `story-writer` → 서사 진단
- `asset-investigator` → 에셋 현황

평가 결과로 다음 10사이클 작업 목록 확정.

## 금지 사항

### 절대 금지

1. **새 곱셈기 추가 금지** — ATK/EXP/Gold formula에 `* newMul` 추가 불가
2. **자동발동 메카닉 추가 금지** — 모든 새 시스템은 플레이어 선택 개입 필수
3. **단일 파일 800줄 초과 금지** — 초과 시 분리가 해당 사이클 작업
4. **10사이클 연속 동일 파일 수정 금지** — 다른 파일도 반드시 건드림

### 프로세스

5. **통합 증명 필수** — 새 코드가 플레이어에게 보이는 경로 증명
6. **Dead code 금지** — export 했으면 import 하는 곳 있어야 함
7. **테스트는 행동 검증** — 존재 확인 테스트 금지

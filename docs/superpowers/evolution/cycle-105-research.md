> 시점의 시장 조사. 2026 idle / incremental / auto-RPG 동향과 inflation-rpg
> 정체성 ("eternal hero idle sponsor", 1 → 수십만 폭발) 정합성 검증.

## 조사 주제

비평가 약점 §1 (signature moment 부재), §2 (player decision frequency),
§3 (long-term retention 엔진 부재) + NEW 방향 N1-N5 **시장 검증** + 2026
동향 기반 **신규 아이디어 1-3 개**. advisor 의 "narrative 외 카테고리 pivot"
권고가 출발점.

## 유사 컨셉 게임 (5 개)

inflation-rpg 정체성 (eternal hero / life cycle / sponsor / 수십만 폭발) 에
근접한 5 개. 단순 idle clicker (Cookie Clicker 등) 는 단발 인용만.

### Hero Generations / ReGen (Steam, 2015 + 2026 ReGen)

- **핵심 mechanic**: 한 걸음 = 1 년. 짝 만나 자식 낳고 죽으면, 그 자식으로
  이어감. 세대 lineage 축적.
  ([Steam](https://steamcommunity.com/app/295590),
  [Presskit](http://www.heartshapedgames.com/press/sheet.php?p=Hero+Generations%3A+ReGen))
- **잘 된 점**: City Crafting — 마을 옆 legacy 건물이 다음 세대로 상속.
  "themes of death, legacy, family, love" — 메커니즘 자체가 narrative 채널.
- **잘 안 된 점**: cult tier 머무름, mass hit 아님. *한 세대의 무게* 가
  가벼워진다는 평. (Steam Charts 수치 직접 확인 안 함 — hedge.)
- **inflation-rpg 적용**: V3 "영원한 영웅" 정체성과 직격 매칭. 현재 단일
  영혼 환생 → *후손 lineage tree* 확장. saga book chapter = 세대 단위 정합.
  Invention A 직접 출처.

### Progress Knight (HTML5, 2021, ihtasham42)

- **핵심 mechanic**: 거지 → 기사 → 마법사 → 신. life 안의 직업 진보, 사망 시
  prestige → xp multiplier. Groundhog Life inspired.
  ([itch.io](https://ihtasham-42.itch.io/progress-knight),
  [Almost Idle](https://almostidle.com/game/progress-knight))
- **잘 된 점**: 생애-as-incremental 의 교과서. age × job 격자가 inflation-rpg
  의 age × realm 격자와 동형. *커뮤니티 mod (Reborn / Quest)* 가 retention
  의 우회 엔진 — live ops 없이도 가능함을 시사.
- **잘 안 된 점**: 시각 자극 0, 글자만. 30분 후 fatigue — critic §1 함정과 동형.
- **inflation-rpg 적용**: 격자 자체는 이미 비슷. *차별화* = N1 (VFX) 로
  Progress Knight 텍스트 함정 회피.

### Lifeidle (itch.io, basigg)

- **핵심 mechanic**: 매 1 분 = 1 년. 선택 누적 → life score → **global
  leaderboard**. ReCycler / Progress Knight 후계작.
  ([itch.io](https://basigg.itch.io/lifeidle))
- **잘 된 점**: self-comparison + 글로벌 비교 동시 — critic §3 (외부 비교 0)
  의 직접 reference.
- **잘 안 된 점**: itch.io 인디. production value 낮음, mass retention 데이터
  부재.
- **inflation-rpg 적용**: N3 (Hall of Sagas + Leaderboard) 직접 시장 검증.
  *local + opt-in 글로벌* 하이브리드가 server-less idle 정체성과 양립.

### Antimatter Dimensions (Web, 2017+, hevipelle)

- **핵심 mechanic**: 다층 prestige — Infinity → Eternity → Reality → ...
  각 층이 게임 룰 재정의. 수십 주~수 개월.
  ([Pinkcrow](https://pinkcrow.net/game-idea/the-best-incremental-games-for-prestige-automation-fans/),
  [Wikipedia](https://en.wikipedia.org/wiki/Incremental_game))
- **잘 된 점**: "numbers so large they require custom notation" — inflation
  정체성의 수학적 best-in-class. maxLevel 6.98M 은 AD 기준 *진입 단계*.
  prestige = narrative beat (2026 트렌드 직접 출처).
- **잘 안 된 점**: 숫자 + 그래프 only, niche audience. *수학 puzzle* 자체가
  보상이라 mass 적용은 위험.
- **inflation-rpg 적용**: inflation *수직 깊이* 확장 모델 참고. 현재 단일
  ascension layer → 2-3 layer 가능. 단 "3 의 규칙" 미충족 — 즉시 적용 X.

### Reventure (Steam, 2019, Pixelatto)

- **핵심 mechanic**: 100 endings 수집 자체가 메인 루프. 매 ending 첫 발견 시
  *다음 플레이 mechanics unlock*.
  ([Steam](https://store.steampowered.com/app/900270/Reventure/),
  [Wikipedia](https://en.wikipedia.org/wiki/Reventure))
- **잘 된 점**: "failure = collectible" — 죽음/실패가 패널티가 아니라 보상.
  inflation-rpg endCause (victory/death/aging 등) 이미 *데이터로 존재* — UI 만
  없음. 매 ending 이 다음 run 의 fingerprint — 시간 축 self-comparison 강화.
- **잘 안 된 점**: 100 endings 채우면 컨텐츠 끝, 무한 retention 안 됨.
- **inflation-rpg 적용**: Invention B 의 직접 출처. inflation-rpg 의 무한 cycle
  정체성과 결합하면 50 endings + 변형 자동 생성 가능.

## 트렌드 / 패턴 (2026)

- **Prestige-as-narrative-beat** — reset 을 *문명적 붕괴, 우주적 재시작,
  진정한 선택* 으로 storytelling. 검증: 6+ 개월 안정.
  [Pinkcrow](https://pinkcrow.net/game-idea/the-best-incremental-games-for-prestige-automation-fans/).
  inflation-rpg 의 ascension + cycle reset 이 이미 이 패턴.
- **하이브리드 장르 확장** — RPG+sim, TD+idle, match-3+narrative. 검증: 6+ 개월.
  [Pinkcrow](https://pinkcrow.net/game-idea/best-idle-games/),
  [Mr.Mine](https://blog.mrmine.com/11-best-idle-games-on-pc-in-2025/).
  inflation-rpg 는 이미 수혜자 — 추가 합치기 보다 깊이 강화.
- **AI agentic content** — background AI 가 player 관찰 + 실시간 수정. 검증:
  3-6 개월 탐색.
  [Bigabid](https://www.bigabid.com/mobile-gaming-trends-2026/).
  **inflation-rpg 부적합** — server-less 정체성 충돌. 안티-패턴 참조.
- **AFK Journey Resonance — auto-share level + 시즌 level 분리** — 한 영웅
  level 상승이 모든 영웅에 자동 적용. 시즌 종료시 시즌 level 소실, 영구 보존.
  검증: 6+ 개월 안정 (메이저 게임 retention).
  [GameLeap](https://www.gameleap.com/articles/afk-journey-resonance-synergy-explained-how-to-unlock).
  적용: V3 "단일 영웅 인생" 정체성과 긴장 — *post-game meta layer* 만 가능.
- **Single-player self-comparison leaderboard** — "single player 의 leaderboard
  는 *자기 자신과의 경쟁* push". 검증: 다년 안정.
  [Gkanatsios](https://dgkanatsios.com/2018/01/08/designing-a-general-purpose-game-leaderboard/).
  N3 의 server-less 변형이 이 패턴.
- **Big-numbers + custom notation** — k/M/B/T/Qa/Qi... 이후 자체 notation. 표준.
  [InnoGames](https://blog.innogames.com/dealing-with-huge-numbers-in-idle-games/),
  [Math of Idle Pt.I](https://www.gamedeveloper.com/design/the-math-of-idle-games-part-i).
  maxLevel 6.98M 은 대중 가독 범위, G/T tier 진입 시 notation 정책 필요.

## N1-N5 시장 검증

### N1 Inflation Milestone VFX
유사 시도: Unity Asset Store Level Up 시리즈, Boris FX VFX guide 모두 milestone
폭발을 표준 기법.
[Unity](https://assetstore.unity.com/packages/vfx/vfx-graph-level-up-effects-vol-1-250944),
[BorisFX](https://borisfx.com/blog/vfx-in-gaming-ultimate-guide-visual-effects-games/).
성공: 부재가 약점 (Progress Knight = 텍스트만 → niche). 차별화: inflation 정체성
의 10^n 폭발 → *10x / 100x / 1000x... 기하 단위 tier*. **검증 완료, 즉시 ship**.

### N2 Mid-cycle Decision Surface
유사 시도: Hades / Hades 2 boon 선택 (Arcana cards).
[Hades 2 Review](https://wolfsgamingblog.com/2026/05/06/hades-2-review-god-tier-roguelike-action/).
*idle + Hades-boon* 교집합 직접 사례 없음. 성공: 액션 roguelike 표준, idle 에선
드뭄. 차별화: idle 에서 *드물게 일어나는 무거운 결정* 으로 포지셔닝. 단 비평가가
추정한 *cycle 당 5-10 회* 빈도는 idle 정체성 깨질 위험 — **scope 축소 권고**
(cycle 당 1-2 회 무거운 결정).

### N3 Hall of Sagas + Local Leaderboard
유사 시도: Lifeidle (life score → global), Hero Generations (legacy 누적).
성공: niche 인디 카테고리 검증, mass hit 는 아니지만 *life-sim incremental
하위 장르 표준*. 차별화: inflation-rpg 는 사가 *데이터 이미 존재* — UI 만 추가.
구현 비용 대비 retention impact 높음. 단 critic §C (외부 비교 0) 를 완전히
해소는 못함 (server-less 한계). **즉시 ship 권고, retention impact medium**.

### N4 Run Statistics View + Curve Visualizer
유사 시도: AD / Exponential Idle 시각화.
[AD Forum](https://forum.quartertothree.com/t/antimatter-dimensions-why-did-this-happen-to-me-first-foray-into-clicker-games/157761),
[Exp Idle](https://namu.wiki/w/Exponential%20Idle).
성공: math idle 하위 장르 필수, 일반 idle 보조. 차별화: cycle 17 polynomial
측정 인프라 이미 존재 — sim 산출과 dev 산출을 같은 차트에 겹치는 dev tool
부산 효과 큼. **scope 작고 부가 가치 큼, headline 은 아님**.

### N5 Live Operation Layer
유사 시도: AFK Journey 시즌 시스템, Melvor expansion DLC (30+ 시간).
[GameLeap](https://www.gameleap.com/articles/afk-journey-resonance-synergy-explained-how-to-unlock),
[Melvor](https://www.incrementaldb.com/game/melvor-idle).
성공: 메이저 idle 의 retention top driver, 단 server-side 일반. 차별화: 비평가
의 *deterministic seed (seasonId = floor(epoch / 30days))* server-less 변형이
정체성 유지하면서 부분 달성 가능. **mega-phase, 즉시 ship 부적합**. Phase 5
(monetization) 결합 시 정체성 혼란 — *시즌 = balance modifier, 결제 X* 의
엄격한 경계 필요.

## 인스피레이션 → invention (3 → 2 신규)

페르소나 룰 3→1 변형 — 사용자 task §4 가 1-3 요구 → 강한 2 개 + 후보 1 언급.

### Invention A — 영혼 환생 lineage

- **inspiration 1**: Hero Generations 의 세대 lineage + City Crafting (legacy
  건물 다음 세대 유전).
- **inspiration 2**: Progress Knight 의 career ladder + age progression.
- **inspiration 3**: Lifeidle 의 every minute = year 시간 압축.
- **invention**: ascension (Phase F-1) 을 *영혼 환생 lineage tree* 로 재해석.
  매 ascension 시 현재 영웅의 *자식 (다음 영혼)* 이 부모의 trait 1 + buff 1
  *유전*. SagaHallScreen → lineage tree visualization 진화. critic §A (체험
  시각화) + §E (saga readability) + V3 정체성 ("영원한 영웅 = 영혼 연속")
  동시 해소. scope: 3-5 cycle. Phase F-1 결합 가능.

### Invention B — Ending Codex

- **inspiration 1**: Reventure 의 *100 endings + 각 ending 첫 발견 시 다음 run
  mechanics unlock*.
- **inspiration 2**: AFK Journey Resonance 의 *영구 vs 시즌 layer 분리*.
- **inspiration 3**: inflation-rpg 자체 endCause 데이터 (이미 존재, UI 없음).
- **invention**: 50 종 unique cycle ending (예: "황금 동굴에서 잠들어 사망",
  "신의 빛에 흡수", "100 세 자연사 → 영웅이 시인이 됨") + 매 ending 첫 발견 시
  *codex 페이지 unlock + 영구 buff 1*. critic §3 (long-term retention) + §A
  (시각화) + §C (외부 비교 0 — codex 가 시간 축 self-comparison 부분 대체)
  동시 해소. server-less 정체성 유지. scope: 3-5 cycle. N3 의 *content 깊이
  axis* 가 됨 — codex 가 leaderboard 의 컨텐츠.

### 후보 C (언급만) — Realm Reshuffle per Ascension

Melvor 의 2026 Cartography ("procedural world map reshuffle per save") 영감.
ascension 마다 6 realm 순서 + 적 비율 셔플. 반복 cycle 신선도 직격. scope mega
— Invention A 의 sub-feature 로 흡수 가능 (lineage 별 realm seed). 주의: Melvor
Cartography 2026 update 출처는 1 차 검색의 요약 모델 진술이므로 spec 단계에서
재검증 필요. [Melvor](https://www.incrementaldb.com/game/melvor-idle).

## 안티-패턴 경고

- **AI agentic content** — Bigabid 2026 트렌드. server-side + 데이터 수집 전제.
  offline single-player + server-less 정체성 충돌. *근본 컨셉 변경* 압력 — 거부.
- **Pay-to-skip / FOMO 시즌 패스** — Adjust / Pinkcrow monetization guide.
  V3 "영원한 영웅 idle sponsor" 의 수동적 관조 정체성 파괴. Phase 5 = *영구
  광고 제거 + 균열석 consumable* 만, 시즌 패스 추가 시 drift. 거부.
- **Hyper-casual + mid-core monetization-driven design** — Adjust 권고 모델.
  디자인 최상위 압력이 monetization 으로 가면 game-design-first 정체성 깨짐.
  현재 Phase 5 별도 layer 분리 구조 유지. 거부.
- **AFK Journey 식 영웅 동시 적재** — 16 캐릭터 동시 사가. V3 "단일 영웅 인생
  사가" 직접 충돌. post-game meta layer 만 가능, base 모델 변경 X.

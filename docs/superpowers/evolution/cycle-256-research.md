> 훑었다면, 256 은 **V3 정체성 자체** (영원한 영웅 idle saga, 6 realm,
> age cycle 5→70+ 자연사 + 회춘, NPC + EternalSaga 통계) 의 인접 영역으로
> 재좌표화한다. cycle 255 carry-over 의 두 mega-phase 중 *EternalCodex*
> (도전록 axis) 는 cycle 156 invention 으로 박제. 256 의 mission = *life-cycle
> axis* 라는 직교 새 axis 1 개를 invention 으로 정제하여, EternalCodex 와
> 충돌 없이 *세 번째 mega-phase 후보* 를 도출한다.

## 조사 주제

V3 의 "영원한 영웅 idle saga" 정체성에 인접한 2025-2026 최신 게임을
4 카테고리에서 살핀다:

1. **Eternal-run / Idle hero rebirth** — Cycle Idle RPG, Reborn, Incremental Epic Hero 2
2. **Lineage / 유전 inheritance** — Noble (lineage 컨셉은 V3 정체성과 충돌, *유전 시스템* 만 추출)
3. **Cosmic / multi-realm prestige** — Antimatter Dimensions, NGU Idle
4. **Anti-pattern 회피** — PANOPTYCA, IdleOn 의 paywall criticism

V3 정체성 (단일 영원한 영웅, server-less, monetization-light) 과 *직접
충돌하는* 모델 (세대 교체 / 결제 게이트 / 시즌 압박) 은 명시적으로 거부.

## 유사 컨셉 게임 (4 개)

### Reborn: An Idle Roguelike RPG (Steam, V1.2.2 = 2025-07, 64% 긍정 / 96 리뷰)

- **핵심 mechanic**: Katie = *immortal warrior*, ever-deepening cave 자동 진입.

  [Metacritic](https://www.metacritic.com/game/reborn-an-idle-roguelike-rpg/),
  [TapTap](https://www.taptap.io/app/33704463))
- **잘 된 점**: "immortal warrior" framing 이 V3 의 *영원한 영웅* 과 직접
  대응. 죽음 = 영구 종료가 아니라 *cycle 의 일부*. unequipped 손실 + XP
  보존의 *비대칭 보존 규칙* 이 player 의 "어떤 trade-off 인지" 즉시 학습
  가능. "to be enjoyed over several months" 의 호흡이 idle 정체성과 합치.
- **잘 안 된 점**: "slow progression, dozens of hours of idling", "tedium
  grows exponentially", sell-all 같은 QoL 부재로 *중후반 chore 화*. 64%
  긍정은 idle 장르에서 미지근 (Incremental Epic Hero 2 의 72%, Antimatter
  Dimensions 의 매우긍정 대비).
- **inflation-rpg 적용 가능성**: V3 의 *자연사 + 회춘* 사이클이 이미 가까운
  framing 이지만, *죽음 자체에 보존/손실 규칙* 이 명시되어 있는지 (V3 의
  현재 회춘 트리거) 는 검증 안 됨. Invention 의 **"death age × realm"
  fan-out 노드** 의 직접 출처. 죽었을 때 *나이 × 어느 realm* 이 다음 cycle
  의 input 이 되는 비대칭 보존 규칙으로 정식화하면, Reborn 의 함정 (느린
  진행) 을 회피하면서 inflation 정체성 (수십만→수백만) 의 *수학적 가속*
  layer 와 합류.

## Anti-pattern 경고 (idle hero / pay-to-idle burnout)

### Anti A1 — Idle 정체성 자기-부정

IdleOn 의 "end-game 은 online constantly 필요" 비판이 보여주는 패턴.
*idle = 자동* 약속을 *후반에 갈수록 active 요구* 로 배신. inflation-rpg
의 V3 정체성 = "player = 신, 관조" — 후반 *수직 폭발 layer* 에 active
input 요구가 들어가면 (예: 영웅 결정 manual override, 정기 사망 시점
선택) 정체성 파괴. cycle 156 의 Anti A2 (daily chore stack) 와 형식 동일,
*hero idle* 영역 변종.
[IdleOn reviews](https://steamcommunity.com/app/1476970/reviews/),
[IdleOn 2026 review](https://trendinggames.gg/games/idleon).

### Anti A2 — Pay-to-idle (기능 핵심을 결제 게이트로)

PANOPTYCA 의 *chest auto-open $8, timer 완화 $14* — *idle 의 본질 기능*
이 paywall. inflation-rpg 의 Phase 5 IAP 4 품목 (광고 제거 non-consumable
+ 균열석 3 tier consumable) 는 *수직 진행 가속* 만 결제, *idle 자동화
자체*는 무료 — anti-pattern 대척점. 미래 IAP 확장 시 "*자동화 자체에
결제* 는 금지" 원칙 박제 필요.
[PANOPTYCA reviews](https://steamcommunity.com/app/3882730/reviews/?browsefilter=toprated).

## 인스피레이션 → invention (3 → 1)

페르소나 룰 준수: 3 inspiration → 1 invention. **합이 아니라 교차** —
3 시스템이 *단일 fan-out 노드* 에서 만나는 형태.

### Invention — "Lifebook" (생애록, life-cycle axis)

- **inspiration 1**: Reborn 의 *immortal warrior + 죽음 = cycle, unequipped
  손실 + XP 보존 비대칭* (T2) — 죽음 시 *무엇이 남는지* 명시.
- **inspiration 2**: Noble 의 *40 gene × 6 allele 유전* (lineage 컨셉은
  **버림**, 유전 mechanics 만 추출) — *과거 N 번 생의 trait pool* 에서
  다음 생 trait roll 의 *유전자 bias*.
- **inspiration 3**: Antimatter Dimensions 의 *layered prestige 의 realm
  별 독립 통화* (T3) — realm 별 영구 multiplier 가 *독립 axis*.

- **invention** — `Lifebook` 이라는 새 axis. **단일 fan-out 노드 = `(death
  age, realm, cause)` 의 3-tuple**. 영웅이 죽을 때마다 이 단일 변수가
  *3 시스템에 동시 fan-out*:
  - (a) **HeroDecisionAI trait roll bias** — 과거 N 번 생의 `(age, realm,
    cause)` pool 에서 *유전자 bias* 추출, 다음 생 trait roll 시 적용 (Noble
    유전 mechanics 의 V3 재프레이밍).
  - (b) **Realm 별 영구 multiplier** — 해당 realm 에서 죽은 *영웅의 나이*
    가 그 realm 의 *독립 통화 (LifeShards 가칭) 적립량* 결정. AD 의
    realm-별-독립-축 변형.
  - (c) **Eternal Saga 영구 narrative entry** — `(age, realm, cause)` 의
    semantic 조합이 *narrative pool 의 한 줄 entry* 로 자동 archive

  256) = *life-cycle axis* (death event → trait pool + realm multiplier
  + narrative). 두 axis 가 *입력 변수가 다르고 (achievement claim vs
  death event), 출력 axis 가 다르고 (수학 multiplier vs trait pool),
  교차 가능* (EternalCodex 의 도전록에 "Lifebook 의 N 번째 entry 도달"
  류 cross-axis 항목 가능, but 그건 미래 cycle). mega-phase 후보 목록에
  *둘 다* 들어가고, 자율진화 시스템이 cycle 256+ 의 약점 핀포인트에 따라
  *어느 axis 가 먼저 우선* 결정.

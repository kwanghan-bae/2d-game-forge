# Cycle 156 웹리서치 (Web Researcher)

> Cycle 155 종료 시점 (SeasonalModifier pure consumer 분할 1/3 완료). N5 Live
> Ops 가 1차 ship 된 직후 — `achievements` (5 starter), `SeasonalModifier`
> (30-day rotation, 8 catalog), `claimerTier` (5 tier 누적 claim), claim
> narration (12×5 = 60 variation) 가 본가에 안착. 본 cycle 의 리서치 mission =
> 2025-2026 시즌 패스 / 도전과제 / live ops 최신 트렌드를 좁혀 핀포인트하고,
> inflation-rpg 가 *놓친* 한 두 트렌드를 식별, anti-pattern 1-2 개를 격리,
> 새 axis 1 개 (이름 + 컨셉) 를 invention 으로 정제.

## 조사 주제

cycle 105 가 idle / incremental 유사 컨셉을 넓게 훑었다면 156 은 그 후 추가된
N5 live ops 의 **시즌 패스 / 도전과제 / 진행 트랙 / burnout 방지** 라는 좁은
질문에 집중한다. inflation-rpg 가 **server-less, monetization-light, 영원한
영웅 idle sponsor** 정체성을 지키면서 2026 live ops 표준에서 *유효한 부분*
만 흡수할 길을 찾는다.

## 유사 컨셉 게임 (4 개)

inflation-rpg 정체성 (idle / eternal hero / sponsor / 시즌 패스 layer) 에 인접한
*도전과제 곱셈자 + 시즌 패스* 라인의 4개. cycle 105 가 idle 광역을 훑었으므로
156 은 N5 (live ops) 영역에 좁혀 골랐다.

### Revolution Idle (Steam, ihtasham42 / 2024+, 다년 운영)

- **핵심 mechanic**: 520 achievements × multiplier bonus + 자원 보상 (Time
  Flux, Souls). 도전과제가 수학적 progression 의 영구 곱셈자.
  ([Wiki](https://revolutionidle.wiki.gg/wiki/Achievements),
  [Guide](https://revolutionidle.wiki.gg/wiki/Guide:Achievements))
- **잘 된 점**: 도전과제가 *체크리스트* 가 아니라 *영구 곱셈자 catalog*.
  500+ 규모로 long-term retention 의 spine. 두 자원 (Time Flux / Souls)
  으로 multi-axis progression 보장.
- **잘 안 된 점**: 520 catalog 의 *작성 비용* 이 거대. 초반 유저 onboarding
  의 grind threshold 가 높음.
- **inflation-rpg 적용 가능성**: Invention "Eternal Codex" 의 *영원의 장* 직접
  출처. 5 starter → 50+ 점진 확장 시 token 의 *영구 ×(1+ε·count)* wire-up.
  inflation 정체성 (수십만→수백만) 의 수학적 곱셈자 layer 와 자연스럽게 합류.

### Idle Dice (Web/Mobile, 다년 운영)

- **핵심 mechanic**: 6 tab 계층 (Basic / Advanced / Expert / Legendary /
  Godlike / Lutslike) 의 도전과제. tab 단위 tier 가 명확한 진행 곡선.
  ([Idle Dice Wiki](https://idle-dice.fandom.com/wiki/Achievements))
- **잘 된 점**: 도전과제 *tier* 명명이 그 자체로 narrative — Lutslike 가
  inflation 정체성과 가까운 hyperbolic naming. inflation-rpg 의 claimerTier
  (신참/노련/숙련/마스터/전설) 가 이미 같은 패턴.
- **잘 안 된 점**: 시각/narrative variance 부재 — 글자 + 숫자 only.
- **inflation-rpg 적용 가능성**: claimerTier 의 5 tier 가 이미 같은 라인.
  Idle Dice 의 *tab 분리* 모델은 EternalCodex 의 *영원의 장 / 계절의 장*
  이중 layer 와 동형 — Invention 의 layer 분리 정당화에 사용.

### Marathon (Bungie, 2026 발표, Reward Pass non-expiring)

- **핵심 mechanic**: Reward Pass 가 시즌 종료 후에도 *영구 진행 가능*. 보유
  pass 의 보상 트랙 = forever, FOMO 차단.
  ([The Escapist 보도](https://www.escapistmagazine.com/bungie-reveals-marathons-reward-passes-never-expire-and-can-be-completed-anytime/))
- **잘 된 점**: 2026 의 새 표준 (Helldivers 2 / Halo Infinite / Marvel Rivals
  동참). 시즌 패스의 FOMO anti-pattern 을 *모델 자체로 무력화*.
- **잘 안 된 점**: 출시 전 발표 단계 (cycle 156 작성 시점 2026-05-27 기준),
  retention 효과의 사후 데이터 부족.
- **inflation-rpg 적용 가능성**: SeasonalModifier 30-day rotation 이 *modifier
  본체는 휘발하되 progress 는 영구 archive* 의 분리 정당화. M1 (영구 vs 시즌
  boundary) 해소의 직접 reference. Invention 의 *시즌 modifier 휘발, 시즌
  도전 항목은 legacy 보존* 의 직접 출처.

### Deep Rock Galactic (Ghost Ship Games, 다년 운영, 시즌 종료 재분배)

- **핵심 mechanic**: 시즌 종료 시 *해당 시즌 보상이 다른 cosmetic pool 로
  재분배* — 시즌 끝나도 컨텐츠 자체는 사라지지 않음.
  ([Happy Gamer Battle Pass Traps](https://happygamer.com/battle-pass-systems-from-seasonal-rewards-to-subscription-traps-138816/),
  [Battle Pass News Models](https://battlepass.news/2025/05/29/battle-pass-models-game-comparisons/))
- **잘 된 점**: *시즌 = 임시 채널, 컨텐츠 = 영구 pool* 의 분리가 player burnout
  방지의 정석. coop 게임 long-tail retention 의 reference 케이스.
- **잘 안 된 점**: cosmetic-only — RPG 식 *수치 progression* 곱셈자 layer 가
  없어 inflation 정체성과 axis 다름.
- **inflation-rpg 적용 가능성**: Invention "Eternal Codex" 의 *계절의 장 →
  영원의 장 archive* 흐름의 직접 출처. 30-day rotation 종료 시 해당 시즌의
  *완료된 도전 항목* 이 lineage book 식으로 영구 보존되는 모델. inflation-rpg
  는 cosmetic 대신 *narrative + 곱셈자* 로 변환.


## 트렌드 / 패턴 (2025-2026 시즌 패스 / live ops)

### Trend T1 — Live Ops 의 monetization → progress + social 축 이동

2026 데이터에서 *monetisation events* 비중이 2025 의 35% 에서 *progress +
social* 의 38% 에 추월당했다. 즉 시즌 패스 자체가 결제 driver 라기보다
*long-term retention + loyalty* 의 spine 으로 재포지셔닝 중. mid-core RPG
template = "8-15 distinct events, 12-24h breathing room between major
competitive events". 검증: 6+ 개월, 메이저 publisher 데이터 기반.
[PocketGamer.biz live ops 2026](https://www.pocketgamer.biz/2026-live-ops-trends-templatisation-personalisation-and-ai/),
[Game Growth Advisor LiveOps Playbook](https://gamegrowthadvisor.com/blog/2026-03-31-liveops-strategy-mobile-games-guide/).

inflation-rpg 의 SeasonalModifier 는 *결제 X, 진행 + narrative axis* 이므로
이 트렌드의 옳은 쪽에 이미 서 있다. 즉 N5 의 좁은 진행이 의도와 일치한다.

### Trend T2 — Non-expiring / "No-FOMO" 패스 모델 부상

Marathon (Bungie), Helldivers 2, Halo Infinite, Marvel Rivals 가 *Reward Pass
영구화* 방향으로 전환. 시즌 종료 후에도 보유 패스의 보상 트랙은 *계속 진행
가능*. Deep Rock Galactic 는 시즌 종료 시 *보상을 다른 cosmetic pool 로 재분배*.
즉 "시즌은 지나가도 컨텐츠는 안 사라진다" 가 2026 의 새 표준. 검증: 4-6 개월,
메이저 발표 일자 기준 (2025 Q4-2026 Q1).
[The Escapist Marathon Reward Pass](https://www.escapistmagazine.com/bungie-reveals-marathons-reward-passes-never-expire-and-can-be-completed-anytime/),
[Happy Gamer Battle Pass Subscription Traps](https://happygamer.com/battle-pass-systems-from-seasonal-rewards-to-subscription-traps-138816/),
[Battle Pass News Models](https://battlepass.news/2025/05/29/battle-pass-models-game-comparisons/).

inflation-rpg 의 SeasonalModifier 는 30-day rotation 이지만 *modifier 본체*
가 종료 시 어떻게 되는지 코드 본 cycle 에서 검증 안 됨 (orientation 만 함).
*무엇이 영구화되고 무엇이 소멸하는지* 의 경계가 디자인 명세로 박혀 있지 않으면
T2 의 함정에 노출됨.

### Trend T3 — Achievement-as-multiplier (Revolution Idle / Idle Dice 형)

Revolution Idle = 520 개 도전과제, 완료 시 *multiplier bonus + 자원 (Time
Flux, Souls)*. Idle Dice = 6 tab 계층 (Basic / Advanced / Expert / Legendary /
Godlike / Lutslike), 완료 시 보너스 points 누적. 즉 도전과제가 단순 *체크리스트*
가 아니라 **수학적 progression 의 영구 곱셈자**. 검증: 두 게임 다년 운영,
500+ 카탈로그 규모, Steam / 모바일 동시 검증.
[Revolution Idle Wiki Achievements](https://revolutionidle.wiki.gg/wiki/Achievements),
[Idle Dice Wiki Achievements](https://idle-dice.fandom.com/wiki/Achievements).

inflation-rpg 의 도전과제 5 starter 는 *token claim* 만, *영구 multiplier*
컨셉 부재. 이게 cycle 156 의 핀포인트 미스 후보 #1.

## inflation-rpg 가 놓친 트렌드 (1-2 개 핀포인트)

### Miss M1 — "영구 vs 시즌" boundary 명시 부재 (T2 derived)

inflation-rpg 의 N5 는 `achievements` (영구 — claim 한 번) + SeasonalModifier
(30일 rotation) + claimerTier (영구 누적) 의 *3 layer 가 한 namespace 에
공존*. 사용자 관점에서 "이거 시즌 끝나면 사라지는 거야?" 질문이 불분명. 2026
표준은 *시즌 = balance modifier (소멸), 보상 트랙 = 영구* 의 명시적 분리.
SeasonalModifier 자체가 trait/narrative/cosmetic axis 라 *gameplay 변형* 인데,
30일 후 *해당 시즌에 누적된 progress* 가 소멸하는지 보존되는지 spec 단계에서
확정 필요. 이대로 가면 T2 의 anti-pattern (gamerant FOMO 치료 사례) 으로
slip 가능.

### Miss M2 — Achievement 의 *영구 곱셈자 hookup* 부재 (T3 derived)

도전과제 claim → token → ??? 의 chain 에서 token 의 다음 효용이 narrative
(claim narration variant) 에만 묶여 있다. Revolution Idle 의 520 achievements
= 각각이 *수학 곱셈자* 인 모델과 대비. inflation-rpg 의 inflation 정체성
(maxLevel 수십만→수백만) 과 도전과제가 *수학적으로* 연결되지 않으면 N5 가
narrative 장식 layer 로 머무름. 5 starter → 50+ catalog 로 확장 시 token →
permanent multiplier (예: ascension multiplier × claim count^0.05) 의 wire-up
이 자연스러운 다음 cycle 후보.

## Anti-pattern 경고 (시즌 패스 / live ops burnout)

### Anti A1 — 만료 압박형 시즌 패스

"몇 일 남음 카운트다운" 으로 grind 강요 → 시간 부족 player 의 *resentment +
burnout + churn*. 2026 단계에서 업계 전반의 *subscription fatigue* 로 확장 중.
gamerant 의 인터뷰는 *"can't dedicate immense hours → frustration → extended
breaks"* 의 chain 을 명시. inflation-rpg 는 *single-player + server-less*
정체성이라 카운트다운 UI 자체가 디자인 위반. 30-day rotation 은 *deterministic
seed (floor(epoch/30d))* 로 player-pull 이 아니라 *world-tick* 으로 표현 권고.
SeasonalModifier 의 30일 rotation 이 *UI 에 timer 로 노출되는 순간* 이 위반
경계.
[GameRant Live-Service FOMO Treadmill](https://gamerant.com/live-service-battle-pass-seasons-fomo-wow-overwatch-2-diablo-4-bad/),
[Unfgaming Burnout 2026](https://unfgaming.net/gaming-fatigue-how-to-recognize-prevent-and-overcome-burnout-in-2026/),
[Prism News Mobile Burnout Design Problem](https://www.prismnews.com/hobbies/mobile-gaming/mobile-game-burnout-is-a-design-problem-experts-say).

### Anti A2 — Daily chore / endless event 스택

PocketGamer Mavens 패널의 합의 = *"burnout is essentially design failure"*.
daily login / 6h 이벤트 / 24h 이벤트 / 주간 이벤트의 *stack* 이 합쳐지면
player 가 "loop the loop" 의무감에 빠짐. inflation-rpg 는 idle sponsor 정체성
(player = 신, 관조) 에 daily quest 류가 *근본 충돌*. SeasonalModifier 에
*daily progress requirement* 가 추가되면 정체성 파괴.
[PocketGamer Mavens Burnout](https://www.pocketgamer.biz/burnout-is-essentially-design-failure-the-mobile-mavens-weigh-in-on-player-burnout/).

## 인스피레이션 → invention (3 → 1)

페르소나 룰 준수: 3 inspiration 1 invention.

### Invention — "Eternal Codex" (영원의 도전록)

- **inspiration 1**: Revolution Idle 의 *520 achievements × multiplier bonus*
  (T3) — 도전과제를 *수학적 영구 progression* 으로.
- **inspiration 2**: Marathon / Helldivers 2 의 *non-expiring reward pass*
  (T2) — 시즌이 지나도 트랙은 영구, 시즌 modifier 만 휘발.
- **inspiration 3**: Deep Rock Galactic 의 *시즌 종료 시 보상 풀 재분배*
  (T2 sibling) — 휘발된 시즌이 영구 pool 에 *합쳐지는* 흐름.

- **invention** — `EternalCodex` 라는 새 axis. 컨셉: inflation-rpg 의
  `achievements` 5 starter 를 **이중 layer 도전록** 으로 진화. (a) *영원의
  장* = T2/T3 합쳐 영구 multiplier 부여하는 catalog (50+ 항목, ascension /
  saga / endCause / 차원별), claim 토큰이 *영구 ×(1+0.005·count)* 의
  inflation-spec 합치 곱셈자로 wire-up. (b) *계절의 장* = 30-day
  SeasonalModifier 의 *완료 항목* 이 시즌 종료 시 자동으로 *영원의 장의
  legacy 페이지* 로 archive 됨 (Deep Rock 식 재분배). 시즌 modifier (gameplay
  변형) 는 휘발, 시즌에 *기록된* 도전 항목은 lineage book 식으로 영구 보존.
  사용자 관점 단일 약속: "당신이 이룬 것은 사라지지 않는다, 다만 세계의
  바람만 달라진다." Anti A1/A2 회피 + claimerTier 누적과 자연스럽게 합류 +
  inflation 정체성 (수학적 폭발) 과 도전과제가 처음으로 *수직 연결*. scope:
  3-5 cycle. cycle 155 의 SeasonalModifier pure consumer 분할이 이미 *시즌
  modifier 를 gameplay 변형으로 격리* 한 토대를 제공하므로 wire-up 자연스럽다.

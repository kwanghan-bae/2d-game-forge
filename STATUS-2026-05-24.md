## 알려진 트랩 (V3-H 추가)

- **`hero_died` event was dead path before V3-H T9**: EncounterEngine 이 hero.staggered=true 만 세팅하고 event 안 emit 했음. T9 가 EncounterEngine 에서 emit 추가 + applyDeathPenalty wire. 만약 트레이드오프 검토 필요하면 (auto-rejuv 비활성 모드 등) `hero_died` emit 자체를 toggle.
- **Cycle persistence vs run persistence**: `useCycleStoreV2.controller` 는 in-memory only (not persisted). `run.heroSnapshot` 만 persist v22 에 보존. App 재진입 시 `start()` 가 snapshot 로드 → controller 재생성. 새 cycle 인지 resume 인지는 `run.heroSnapshot !== null` 로 판단.
- **Season bonus dampingThresholdBonus**: winter 에서 buff #6 threshold +5. `cycleSliceV2.getBuffSnapshot` 의 damping 계산에 직접 추가. V3-G balance 시 winter 의 damping 효과 재검토 필요 (다른 season bonus 와 magnitude 비교).
- **HeroEntity.serialize 의 Set 직렬화**: `Set<JobMilestone>` + `Set<string>` 을 array 로 직렬화 후 restore 시 Set 재생성. 새 Set 필드 추가 시 serialize/restore 양쪽 갱신 필요.
- **자동 회춘 setTimeout 2초**: OverworldRunner 의 `autoRejuvTimerRef` 가 unmount 시 정리. 컴포넌트가 마지막 cycle 끝나기 전 unmount 되면 rejuv 안 일어남 — 의도된 동작 (메인 메뉴로 나가면 cycle 멈춤).

## 알려진 트랩 (V3-DEF 추가)

- **Controller purity drift V3-E/F**: V3-D 는 callbacks 로 `CycleControllerV2` 순수 유지했지만 V3-E (T19/T20) + V3-F (T25) 는 `useGameStore` 를 controller 안에서 직접 호출. 의도된 deviation (NPC + saga 는 sim 무관). V3-G 또는 별도 refactor phase 에서 purity 재정립 또는 수용 결정 필요.
- **Damping floor `Math.max(1, ...)`**: T11 의 hero atk damping 계산이 `Math.max(1, Math.floor(hero.atk * damping))` — plan 외 defensive. `floor(0.4)=0` 무한루프 방어. V3-G balance pass 에서 magnitude 변경 시 floor 가 결과를 왜곡하지 않도록 주의.
- **persist v21 = v20→v21 idempotent** — `migrateV20ToV21` 가 4 필드 주입. v21 → v22 추가 migration 작성 시 같은 패턴 따라 작성.
- **EternalSaga eraKeyFor**: rejuvCount 0 = `'본래 X'`, > 0 = `'재생 #N X'`. SagaBookModal 의 era sort 가 rejuvCount 우선 → chapter order 보조. 새 chapter 추가 시 sort order 배열 (`['어린시절','청년기','장년기','노년기','마지막']`) 갱신 필요.
- **NPC tick + zustand mutation**: T19 의 `tickNpc(npc)` 가 store snapshot 의 npc 객체를 in-place mutate. 다음 set() 까지 subscriber 미감지. encounter 이벤트 emit 으로 우회되지만 정밀 UX 필요 시 별도 store dispatch 필요.

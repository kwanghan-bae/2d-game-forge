## 2. 알려진 limit (V1c-2 finding 포함)

### 2a. atk-bound dead zone — V1c-2 시도 후 구조적 finding

`docs/superpowers/reports/2026-05-22-phase-v1c-2-abandoned.md` 의 상세. 핵심:

- **곡선 exponent 는 maxLevel 에 literally 무관**. k_eHp 0.8 / 1.0 / 1.15 의 sim 측정 maxLevel P50 = 14,268 / 14,162 / 14,162 (< 1% 변동).
- multi-scenario 4 strategy 별 last-cycle maxLevel = 13,859 동일 (criterion #6 strategy 차이 = **0%**).
- Root cause: `BP cost = 1 per kill (flat)` × `level-up → hp = hpMax (full heal)`. Hero 가 매 kill 마다 76 level-ups → 76 full heals → 사실상 불사 → BP-bound (BP=100 → 100 kills → fixed maxLevel).

**user 가 결정할 두 design 갈래:**

1. **BP cost ∝ swings** — `EncounterEngine` 가 `ceil(enemyHp / heroAtk)` BP 소비. atk 가 swing 수를 줄여 BP 절약 → cycle 길어짐 → maxLevel 증가. atk-bound 즉시 surface.
2. **Level-up heal cap/제거** — 76회 full heal 가 사라지면 hero 가 죽을 수 있음. enemy variance / boss 다단 retaliation 의 balance 가 의미. atk-bound 의 다른 면.

둘 다 user-facing balance 변화. 자율 세션이 결정할 범위 밖. 어느 방향이든 결정 후 `Phase V1d` 또는 `V1c-3` 으로 진행 가능.

### 2b. Saint 58.5% 가 새 dominant fallback

V1c-1 의 결과: final job 분포 `saint 117 / dark_lord 31 / archmage 23 / grandmaster 15 / sage 8 / hero 6`. sage 99% → saint 58.5%. criterion #1 의 글자 그대로 (sage <50%, distinct ≥6) 는 통과지만 "한 직업 지배" 패턴은 그대로.

Root cause: `battle_won merciful proc 15%` 가 cycle 당 enemy kill 100+ 회 × 15% = ~14회 fire — 가장 많은 drift surface. merciful 이 가장 빠르게 +8 도달 → saint unlock 가장 흔함.

**의도된 design 인지 V1c+ tuning target 인지 user 결정 필요.** 만약 tuning target 이면:
- merciful proc rate 5-7% 로 낮춤
- 또는 다른 dim 의 drift surface 추가 (예: 각 새 landmark 별 instance 수 ↑ 3개)
- 또는 tier-3 unlock 의 threshold 분포 평준화

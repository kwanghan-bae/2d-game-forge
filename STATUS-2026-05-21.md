## 2. 알려진 limit / 의식적으로 punt 한 것

### 2a. UI 작업 모두 미룸

V1b 의 T4 (job-unlock cinematic toast), T11 (saga book viewer), T13 (e2e) 는 browser 검증이 필요해서 autonomous 에서 안 함. 데이터 / 백엔드는 다 있다. UI 만 얹으면 작동.

`docs/superpowers/plans/2026-05-21-phase-v1b-job-skill-shrine-morality-saga.md` 의 T4 / T11 / T13 task 그대로 사용 가능.

### 2b. Tier-3 직업 다양성 부족

200 cycle 표본에서 final job 이 거의 모두 sage (fallback). dark_lord 2개만 unique. 이유: HeroSpawner 가 priors 를 ±[1,5] 로 주고, cycle 내 personality drift 는 moral 만 ±2 → tier-3 threshold (heroic≥8 / pious≥6 / merciful≥7 등) 까지 못 자람.

**해결 방향:** V1c 에서 personality-drift 인카운터를 더 추가 (각 dim 별 cave/ruin 유사 분기). Spec §7 의 Tier 2 "도덕 분기 확장 (10+ encounter type × 5 dim)" 이 정확히 이 작업.

### 2c. Meta-progression bonus 가 maxLevel 에 영향 X

Multi-scenario sim 의 비교 표를 보면 strategy (balanced/atk-focus/hp-focus/personality) 별 maxLevel 가 동일 (cycle별로 동일 seed → 동일 outcome). 이유:

현재 곡선에서 hero.atk = atkBase × lv^1.0 이 enemy.hp = 30 × lv 보다 항상 ~1.67x → hero 가 매번 1-hit kill → enemy retaliation 0 → atkBase 보너스가 추가되어도 1-hit 그대로 → 어차피 1 kill = 1 BP → BP-bounded 사이클 길이 동일 → maxLevel 동일.

**즉 현재 V1a 의 게임은 BP-bound 이지 atk-bound 가 아니다.** sponsorGold 가 "느낌은" 있지만 (영구히 누적되는 hp/atk 보너스를 표시) 실제 cycle outcome 에는 영향이 적다.

**V1c balance 의 큰 작업:** atk 가 부족하면 hero 가 죽을 수 있는 시나리오 도입. 예:
- Enemy hp 천장 (현재 lv^1.0 → lv^1.1 로 살짝 빠르게)
- Boss 가 정말 위협적 (탱크 + 다단 retaliation)
- 죽음 위험을 도입해서 atk/hp 보너스가 의미 있게

이 작업은 advisor 가 시즌 5의 priority 4 로 언급한 "RNG axes 추가 + 메타 progression 으로 cap 확장" 의 연속.

### 2e. E2E vertical-slice 가 느려졌다 (bpMax 30→100 의 부작용)

### 2d. 10,000 시간 컨텐츠 깊이 — 정직한 평가

**Substrate (지금):**
- 16 jobs × ~21 skills (각 hero 가 7-10 학습) × 5 personality dims × ~30 equipment × moral choice variations × 14K level inflation
- 거친 추정: 1 cycle ≈ 5-10 분 (sim time) × ~수만 가능한 cycle 조합 = **약 1,000-5,000 시간 변동가능 컨텐츠**

**Substrate 가 안 주는 것 (V1c 이상 필요):**
- 라이벌 / 멘토 NPC (persistent across cycles) — V2 spec Tier 2
- 가족 / 결혼 / 자식 — Tier 3
- 큰 사건 / 전쟁 / 정변 — Tier 3
- 명성 / 악명 — Tier 3
- 환생 / 가문 영지 — Tier 4
- 균열 (Compass legacy 재활용) — V1c

**10K 시간 도달 시뮬레이션:**
1. V1b balance fix → atk-bound 활성 (1 → 1.5K hours)
2. V1c: 라이벌 + 멘토 + 균열 + UI → 5K hours
3. V2 Tier 2-3 (가족 / 사건 / 명성 / 환생) → 15K hours
4. V2 Tier 4 (가문 / 영지 / 종교) → 30K+ hours

즉 **10K 시간은 V2 Tier 2 ~ Tier 3 의 narrative depth 가 들어가야 도달한다**. 현재는 그 substrate 만 있다 — 무한 가까운 cycle outcome 의 _가능성_ 은 있지만 _깊이_ 는 더 콘텐츠 작업 필요.

**Task breakdown — 자율 세션이 advisor 와 상의 + 자체 plan 후 진행:**

#### A. Personality-drift 인카운터 (각 dim 균등 coverage)

현재 cave (heroic+merciful → moral) + ruin (moral → moral) 만 있음. moral 외 dim 은 priors 가 가능한 max ±5 에서 cycle 내내 멈춤 → tier-3 threshold (±6~8) 도달 불가 → sage fallback.

추가할 dim 별 인카운터 (각 dim 별 ≥ 2 trigger, ±2-4 drift):
- **heroic**: 전투 동참 vs 회피 (예: 마을 습격 광경, 동료 위기)
- **prudent**: 위험한 보물 vs 안전한 길 (예: 수상한 동굴, 신비의 약초)
- **merciful**: 적 자비 vs 처치 (battle_won 직후 random 모달, 잡몹 한정)
- **pious**: 사당 깊은 기도 vs 통과 (shrine 확장 또는 신성 폐허)
- **moral**: 추가 분기 (cave/ruin 외 2-3 종)

신규 파일: `src/data/personalityEncounters.ts` (catalog) + `src/overworld/EncounterEngine.ts` wire. 새 landmark kind 추가 (`watchtower` / `treasure_cave` / `holy_ruin` 등) + `mapLayout.ts` placement 변경.

#### B. Balance fix — atk-bound dead zone

현재 hero.atk = atkBase × lv^1.0 = enemy.hp 와 동일 ratio → 모든 enemy 1-hit kill → atk 보너스가 maxLevel 에 영향 X.

`src/cycle/inflationCurve.ts`:
- `k_eHp: 1.0 → 1.1` (enemy hp 가 hero atk 보다 살짝 가파르게)
- `BOSS_HP_MUL: 4 → 5-6` (boss 가 multi-hit 필요)
- `BOSS_ATK_MUL: 2 → 3` (boss retaliation 위협)

Sim-G round 으로 재검증 — success bar 1-4 모두 통과해야. 안되면 exponent 미세 조정.

#### C. RNG axes 추가 (advisor priority 1 — Sim-G baseline §F2 finding)

`src/overworld/EncounterEngine.ts`:
- **Crit roll**: 10% 확률로 hero.atk ×2 (high heroic → 15%, high prudent → +crit damage)
- **Drop tier roll**: drop 시 common 70% / rare 25% / epic 5% (현재는 ENEMY_DROPS 균등). Boss drop 은 rare 60% / epic 30% / legendary 10%
- **Enemy variance**: hp/atk 에 ±20% noise (SeededRng)

모든 RNG 가 `EncounterEngine.rng` (SeededRng) 사용 — determinism 유지. legendary drop 필요 시 `dropTable.ts` 에 +20-40 atkFlat tier 추가.

#### D. Sponsor gold auto-spend

`src/overworld/cycleSliceV2.ts` 의 endCycle 에서 'balanced' strategy 로 자동 spend. 또는 별도 `applyAutoSpend()` 함수. CyclePrepV2 는 "자동 후원: ATK+N HP+N (총 N gold 소비)" 표시.
